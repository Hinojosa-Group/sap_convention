📖 SDK ABAP para Cloud Storage 📖

Detalles de Implementación como Content Repository SAP ArchiveLink

⚙️ Sustitución del Content Server Tradicional

La integración certificada permite que Google Cloud Storage (GCS) actúe directamente como un repositorio oficial de contenido de SAP bajo el estándar de SAP ArchiveLink.

Esto se logra mediante la instalación y configuración del SDK de Google Cloud para ABAP directamente en el servidor de SAP (S/4HANA o NetWeaver), eliminando servidores intermedios de bases de datos y middleware de terceros (como OpenText o servidores KPro locales).

A continuación se detallan los aspectos clave de esta arquitectura técnica:

  • Comunicación Nativa y Certificada: SAP realiza peticiones HTTP directas hacia las APIs REST de Google Cloud Storage utilizando protocolos HTTPS cifrados e integrados en el Kernel de SAP (con la transacción OAC0).
  • Autenticación Segura (OAuth 2.0): El SDK de ABAP gestiona la obtención, refresco y almacenamiento seguro de los tokens de acceso utilizando perfiles de autenticación en la transacción /GOOG/CLIENT_KEY, securizados a través del Almacenamiento Seguro (Secure Storage) de SAP.
  • Cumplimiento de ArchiveLink: Soporta todas las operaciones estándares del protocolo ArchiveLink (creación, lectura, borrado y modificación de objetos) mapeando de forma automática las llamadas RFC de SAP a peticiones HTTPS REST en GCS.
  • Ahorro de Costes e Infraestructura: Elimina las licencias y el coste operativo de mantener bases de datos relacionales dedicadas únicamente a adjuntos. Se paga estrictamente por gigabyte almacenado en GCS con una durabilidad del 99.999999999% (11 nueves).

💻 Transacciones Clave de Configuración en SAP

Para la puesta en marcha de esta solución, se configuran las siguientes transacciones en el backend de SAP:

* /GOOG/SDK_IMG - Guía de Configuración e Implementación del SDK de Google Cloud * /GOOG/CLIENT_KEY - Registro seguro de la cuenta de servicio y alcances (scopes) OAuth * OAC0 - Definición del Content Repository de SAP (Tipo: HTTP Content Server) * OAC3 - Enlace de los tipos de documento y objetos de negocio con el repositorio
