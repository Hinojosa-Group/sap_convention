📦 1. Nuevo Servidor de Contenido: Google Cloud Storage

Capa de Almacenamiento

Adiós a los servidores locales lentos e inestables que se quejan cada vez que un albarán pesa más de un mega. Hemos migrado la capa de almacenamiento de contenido de SAP a Google Cloud Storage (GCS) de forma nativa e integrada.

1.1 Sustitución de ArchiveLink mediante el Add-On de Google

Hemos implementado el add-on certificado de Google para SAP que sustituye el tradicional SAP Content Server por Google Cloud Storage. La gran ventaja es que la integración es 100% nativa utilizando el SDK de ABAP, lo que permite a SAP comunicarse directamente con el bucket mediante llamadas certificadas de ArchiveLink.

Esto reduce la complejidad de la infraestructura, elimina el mantenimiento de bases de datos locales dedicadas a documentos y reduce drásticamente el uso de espacio en la base de datos de SAP.

📚 Explicación Técnica Detallada del Add-On ABAP GCS ➔

1.2 Publicación de Objetos del Bucket Directamente (URLs Anónimas)

Para optimizar la experiencia de usuario y agilizar los sistemas de información, implementamos la publicación de objetos del bucket directamente como URLs anónimas seguras, evitando sobrecargar el servidor SAP.

A través del API Hub (Viewer Service), el sistema genera de forma automática enlaces securizados e independientes. Este servicio actúa como un proxy desacoplado: recibe las llamadas, autentica server-to-server contra el bucket, recupera el PDF de la carpeta definitiva y lo transmite al cliente final de forma transparente, sin exponer credenciales nativas de GCP.

De este modo, los usuarios de SAP pueden consultar la documentación en tiempo real mediante un navegador simple con un rendimiento de milisegundos.

👻 3. El Proyecto Estrella: GOS-Busters: The Ingestion Protocol 👻

AUTOMATED AI WORKFLOW

Ha nacido GOS-Busters 👻, nuestro sistema estrella serverless y orientado a eventos que automatiza la ingesta masiva de PDFs de SAP utilizando Inteligencia Artificial Multimodal (Gemini 1.5 Flash). Su arquitectura es 100% serverless, desacoplada por eventos e idempotente.

Para que el equipo técnico comprenda al detalle cómo funciona la orquestación bajo el capó, desglosamos los dos casos de uso principales implementados con sus respectivos diagramas de secuencia:

🛒 Caso de Uso 1: BUS2032 (Pedidos de Venta)

Este flujo procesa archivos PDF de pedidos de venta completos. Gemini 1.5 Flash extrae un payload JSON estructurado con la información comercial (cliente, referencia, posiciones, etc.) que se enriquece y envía a SAP para la creación del pedido en firme.

📄 Caso de Uso 2: LIKP (Albaranes de Entrega)

Este flujo procesa documentos PDF de albaranes multipartición. El PDF original se divide en páginas en memoria, Gemini extrae visualmente el número de documento de cada página, se localiza en SAP y, tras verificar que la URL anónima es accesible, se adjunta el link y el binario correspondiente al objeto de negocio en SAP GOS.

🚀 Posibilidades de Expansión de la Plataforma

  • Auto-Clasificación de Documentos: Dejar caer cualquier archivo (JPG, PDF, PNG) en una carpeta genérica y que Gemini decida automáticamente si es un Albarán, Factura o Ficha de seguridad, y dispare el flujo correspondiente.
  • Extracción Inteligente de Incidencias: Leer correos técnicos de soporte SAP, extraer el error del pantallazo de SAP GUI usando visión artificial, tipificar el error de forma automática, y abrir el ticket directamente asignado en Jira.
  • Mantenimiento Predictivo Serverless: Analizar los logs de BigQuery y dumps de SAP en tiempo real para predecir si un servidor de CPI está a punto de quedarse sin memoria, de modo que se puedan escalar recursos de forma automatizada.
