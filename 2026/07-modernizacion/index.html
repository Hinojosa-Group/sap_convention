📂 GOOGLE CLOUD STORAGE CONTENT SERVER Almacenamiento Infinito

Se acabó pagar cuotas millonarias por almacenar PDFs pesados de facturas y albaranes en la base de datos de SAP. Al migrar a Google Cloud Storage (GCS) como nuestro Content Repository principal, ganamos escalabilidad elástica y abaratamos costes a niveles ridículos.

¿Qué ganamos con el nuevo Content Server en GCS?

  • Ahorro del 95% en costes de almacenamiento de base de datos SAP.
  • Alta Disponibilidad: Los documentos están disponibles 24/7 de forma redundante.
  • Acceso Seguro y Controlado: El visor de documentos no accede directamente a la URL nativa de Google, sino que pasa por un proxy federado en nuestro API Hub que valida el acceso de forma segura.

Anexo 1: Publicación y Verificación de Accesibilidad

Para asegurar que los documentos estén listos y disponibles públicamente para los usuarios en el GOS de SAP sin comprometer la seguridad de las credenciales de nuestro bucket, el componente LikpHandler ejecuta el siguiente flujo transaccional y robusto:

Nombre final: {centro}_{doc_num}_{entropia}.pdf H->>GCS: move_file(20.processing/..., 30.ready/...) GCS-->>H: Confirmación de reubicación exitosa Note over H: Construye url_final:

la URL y el binario en SAP GOS

Detalle Técnico de los Pasos:

  • 1
    Persistencia Definitiva (30.ready): El archivo PDF es movido desde el espacio temporal `20.processing/` a su ubicación inmutable en `30.ready/{tipo_documento}/...` renombrado con un identificador unificado y una entropía de seguridad criptográfica.
  • 2
    Construcción de URL del Visor: Se genera una URL federada de API Hub (`IS_VIEWER_SERVICE`) en lugar de apuntar directamente al bucket de Google Cloud. Esto actúa como un proxy desacoplado que no expone la estructura interna de almacenamiento.
  • 3
    Validación de Accesibilidad (Health Check): LikpHandler realiza un GET simulando al usuario final. API Hub se conecta mediante Service Account a GCS, descarga el binario en memoria y lo transmite. Si el HTTP status es 200, se confirma que la accesibilidad es correcta y se procede a enlazarse en el GOS de SAP.

La herramienta zarchivelink_export (también conocida en clave militar como GOS-Buster Explorer) permite a los técnicos y planificadores explorar visualmente todos los adjuntos enlazados mediante Generic Object Services (GOS) o ArchiveLink directamente desde SAP, de una forma rápida e intuitiva.

Pantalla 1: Filtro de exploración y selección de documentos GOS/ArchiveLink en SAP.

🤖 MOTOR DE INGESTIÓN: HPG DOC-INHALER Pro Powered by Gemini

Aquí presentamos la joya de la corona de la automatización documental: **HPG DOC-INHALER Pro** (el servicio standalone de ingesta estructurada descrito en el Anexo 2). Este microservicio en Python corre en Cloud Run guiado por eventos, devorando PDFs y subiéndolos directamente a SAP.

Dashboard de Control: Monitorización en tiempo real de albaranes y pedidos procesados por el motor de IA.

Flujo de Trabajo del Motor

La arquitectura se basa en Handlers especializados e independientes para cada flujo de documento (idempotencia garantizada):

  • 1
    Recepción en 10.in/: Un nuevo PDF entra en la bandeja de entrada del bucket de GCS (`10.in/LIKP/` o `10.in/BUS2032/`). Eventarc dispara el webhook en Cloud Run de inmediato.
  • 2
    Extracción Inteligente (Vertex AI): Gemini 1.5 Flash analiza el PDF visualmente y extrae los metadatos de negocio críticos (ID, fechas, posiciones) formateándolos a JSON estricto en milisegundos.
  • 3
    Publicación y Enlace en SAP: Se valida la existencia del objeto de negocio en SAP y se enlaza el PDF usando ArchiveLink/GOS. El archivo procesado se archiva de forma inmutable en `30.ready/`.

🚀 Pedidos_IA (BUS2032): Creación Automática de Pedidos de Venta

Basado en el motor de GOS-Buster y el DOC-INHALER Pro, el flujo de BUS2032 va más allá del simple archivado: procesa PDFs de pedidos de clientes, extrae posiciones completas con Gemini y envía un payload JSON estructurado directamente a SAP para la creación automatizada de pedidos de venta. **¡Cero esfuerzo manual, cero errores de transcripción!**
